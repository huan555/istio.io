本页面介绍了 Istio 支持的可用于控制系统操作的各种资源标签 。

标签名称 功能状态 资源类型 描述
istio.io/rev Alpha [命名空间] Istio控制平面版本与资源相关联; 例如，“canary”
networking.istio.io/gatewayPort Alpha [服务] IstioGatewayPortLabel 覆盖了默认端口值的 15443，用于指定多网络网关的端口使用情况。
service.istio.io/canonical-name Alpha [Pod] 工作负载所属的标准服务的名称
service.istio.io/canonical-revision Alpha [Pod] 工作负载所属的规范服务中的版本名称
sidecar.istio.io/inject Beta [Pod] 指定是否应自动注入 Envoy sidecar 到工作负载中。
topology.istio.io/cluster Alpha [Pod] 这个标签是集群内部应用于工作负载的一个标识，用来识别包含该工作负载的 Kubernetes 集群。在Istio安装过程中，通过 values.global.multiCluster.clusterName 为每个集群指定了集群ID。需要注意的是，这个标签仅在 Istio 内部使用，并不是工作负载 Pod 上的实际标签。如果一个 Pod 包含了这个标签，它会被 Istio 内部根据Istio安装时指定的集群 ID 所覆盖。这个标签提供了一种通过集群选择工作负载的方式，当使用 DestinationRule 时很有用。例如，一个服务所有者可以创建一个包含每个集群服务版本的DestinationRule，然后使用这些服务版本来独立地控制流向每个集群的流量。
topology.istio.io/network Beta [命名空间 Pod 服务] 用于标识一个或多个Pod所属网络的标签。这个标签在 Istio 内部用于对属于同一 L3 域/网络的 Pod 进行分组。Istio 假设在同一网络中的 Pod 可以直接相互通信。当 Pod 位于不同网络时，通常使用一个 Istio 网关（例如：东西向网关）来建立连接（使用 AUTO_PASSTHROUG H模式）。这个标签可以应用于以下资源，用于帮助自动化配置 Istio 的多网络环境。

* Istio 系统命名空间：将此标签应用于系统命名空间可以为控制平面管理的 Pod 设置一个默认网络。通常在控制平面安装过程中，由管理员配置该值。

* Pod：将此标签应用于 Pod 可以在每个 Pod 上覆盖基本的默认网络。通常通过 Webhook 注入将此标签应用于 Pod，也可以由服务所有者在 Pod 上手动指定。每个集群中的 Istio 安装使用管理员提供的值配置 Webhook 注入。

* 网关服务：将此标签应用于 Istio 网关的服务，表示 Istio 应该将此服务用作配置跨网络流量的网关。Istio 将配置位于网络之外的 Pod 通过 spec.externalIPs、status.loadBalancer.ingress[].ip 访问网关服务，或在 NodePort 服务的情况下通过节点的地址访问网关服务。此标签在创建网关时进行配置（例如：东西向网关），应与控制平面的默认网络（由 Istio 系统命名空间标签指定）或目标 Pod 的网络匹配。
topology.istio.io/subzone Beta [节点] 用户提供的节点标签，用于标识工作负载的分区。这使管理员可以指定比 Kubernetes 的地区和区域默认提供的更细粒度的地域性。
